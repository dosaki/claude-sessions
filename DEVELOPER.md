## Architecture

The tool supports three execution modes, all sharing the same **discover → classify → render** pipeline:

```
                                    ┌─────────────┐
                                    │  GUI (Fyne) │  ← default
                                    │  gui.Run()  │
                                    └──────┬──────┘
                                           │
┌──────────────────┐     ┌─────────────────┤     ┌──────────────┐
│  GetClaudeProcs  │────▶│ ClassifySessions│────▶│  CLI Draw()  │  ← --cli
│  (platform_*.go) │     │ + Filter + Sort │     │  (render/)   │
└──────────────────┘     └─────────────────┤     └──────────────┘
                                           │
                                           │     ┌───────────────┐
                                           └────▶│  One-shot     │  ← --once
                                                 │  (render/)    │
                                                 └───────────────┘
```

Each session's enrichment (WorkDir, Source, JSONL state) runs in a parallel goroutine since all three calls involve I/O (process lookups, file reads).

### Package Dependency Graph

```
cmd/main ──▶ ansi
         ──▶ config
         ──▶ discovery
         ──▶ focus    ──▶ discovery
         ──▶ gui      ──▶ config
                      ──▶ discovery
                      ──▶ focus
                      ──▶ util
         ──▶ terminal ──▶ ansi
         ──▶ render   ──▶ ansi
                      ──▶ config
                      ──▶ discovery
                      ──▶ terminal
                      ──▶ util
         (util       ──▶ discovery)
```

No circular dependencies. `ansi`, `config`, and `discovery` are leaf packages with no internal imports. `util` depends only on `discovery` (for `HomeDir`).

### Key Design Decisions

**Three execution modes** — GUI (Fyne) is the default because it provides richer interaction (click-to-sort, click-to-focus, filter dialog). `--cli` falls back to the ANSI terminal dashboard for SSH/headless use. `--once` is a non-interactive one-shot print for scripting.

**Fyne GUI** — The `gui` package uses Fyne v2's `widget.Table` for the session list. Column headers are flat buttons that toggle sort on click. Row selection triggers `focus.FocusSession()`. A background goroutine refreshes data on a timer and updates the UI via `fyne.Do()`.

**Session focusing** — The `focus` package activates the terminal/IDE running a given session. On macOS, it uses AppleScript to find Terminal.app/iTerm2 windows by TTY or window title, and walks the process tree to identify the terminal emulator. For IDEs (VSCode, Cursor, Windsurf), it invokes their CLI tool. tmux sessions are focused by pane PID lookup.

**Windows console attach** — On Windows, GUI apps don't have a console by default. `console_windows.go` calls `AttachConsole` so that `--cli` and `--once` modes can write to the parent terminal.

**Concurrent enrichment** — Each session's working directory, source, and JSONL state are resolved in parallel goroutines (`ClassifySessions`). These involve I/O-bound calls (`lsof`, `ps`, file reads), so concurrency significantly reduces render latency when many sessions are active.

**JSONL-based status** — CPU usage is checked first: above threshold means "Working" regardless of JSONL state. When CPU is low, the last entry of Claude Code's session log (`~/.claude/projects/<encoded-workdir>/<uuid>.jsonl`) is read to distinguish "waiting for tool approval" from "idle after completion":

| Condition (evaluated in order)     | Status  |
|------------------------------------|---------|
| CPU > threshold                    | Working |
| JSONL: `assistant` + `tool_use`    | Waiting |
| Anything else                      | Idle    |

**ANSI-aware rendering** — The `ansi` package handles escape codes when measuring string widths and truncating to terminal width. `VisibleLen()` strips escape sequences before counting runes; `Truncate()` preserves escape context while truncating and appends a reset.

**Flicker-free updates** — Instead of clearing the screen each frame, the CLI renderer uses cursor-home (`\033[H`) and clear-to-end-of-line (`\033[K`) to overwrite in place.

## Building

```bash
# Build binary
make build

# Run all tests
make test

# macOS .app bundle (macOS only)
make app

# Cross-compile for Windows (requires CGO cross-compiler)
make build-windows

# Generate macOS icon from SVG
make icon

# Clean build artifacts
make clean
```

Fyne requires CGO for OpenGL bindings. The standard `go build` and `go run` work for native builds. Cross-compilation to Windows requires a C cross-compiler (e.g., mingw-w64) or use `make build-windows` which sets the appropriate flags.

## Common Development Tasks

### Adding a New IDE/Source

1. **Command-line detection** (shared across platforms): add an entry to `idePatterns` in `internal/discovery/filter.go`
2. **Parent process detection** (platform-specific): update the switch in `GetSource()` in `platform_unix.go` and/or `platform_windows.go`

### Adding a New Column

1. Add the field to `Session` in `internal/discovery/classify.go`
2. Populate it in `ClassifySessions()` (add to the goroutine if I/O-bound)
3. **CLI**: adjust `fixedColumnsWidth` and row format in `internal/render/render.go`
4. **GUI**: add column index constant, header, cell rendering, and width in `internal/gui/gui.go`

### Adding Sort Support for a New Column

1. Add the column name to `ValidSortColumns` in `internal/discovery/sort.go`
2. Add a case in the `SortSessions` switch in the same file
3. Add to `columnSortKeys` in `internal/gui/gui.go`

### Adding Focus Support for a New Terminal

1. Add the terminal to `terminalPatterns` in `internal/focus/focus_unix.go`
2. If it needs special activation logic, add a dedicated function and case in `FocusSession()`
3. For Windows, update `internal/focus/focus_windows.go`

### Changing Status Logic

Modify `DetermineStatus()` in `internal/discovery/classify.go`. This is a pure function of `(cpu, cpuThreshold, sessionState)` — easy to test in isolation.

### Modifying the GUI Theme

Edit `internal/gui/theme.go`. The theme uses a Catppuccin-inspired dark palette. Color, font, and size overrides are in the `dashboardTheme` methods.

### Adding ANSI Helpers

Add to `internal/ansi/ansi.go`. This is a leaf package with no internal imports, so changes here won't cascade.

## Platform Notes

| Platform | Process discovery | CWD resolution | Terminal resize | CPU status | Session focus |
|----------|------------------|----------------|-----------------|------------|---------------|
| macOS    | `ps -eo`         | `lsof -p`      | SIGWINCH        | Yes        | Terminal (AppleScript), IDE, tmux |
| Linux    | `ps -eo`         | `/proc/<pid>/cwd` readlink | SIGWINCH | Yes     | IDE, tmux     |
| Windows  | PowerShell `Get-CimInstance` | PowerShell (best-effort) | Polling (500ms) | No (JSONL only) | IDE only |
